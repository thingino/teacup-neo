Notes:
- 2 GPIOs genuinely spare (29, 35); everything else allocated.
- Pins 28-30 (SPIIO6/SPIIO7/SPIDQS) are octal-PSRAM-only pins,
  usable as plain GPIO here because this is the quad-PSRAM (R2) variant.
- EN_CS_U4/U5 (flash CS-select) and SW5V_EN_BMC/SW5V_EN_ALT (power-domain
  switching) are kept on native GPIO deliberately -- margin/timing testing.
- USB_OC1-3 kept native for fast overcurrent response.
- USB_EN1-3, PG_SW5V, PG_VCORE, PG_SW5V_ALT, SW_SENSE_1/4, DCJACK_PRESENT
  moved off native GPIO onto U15 (TCA9555PWR, 16-bit I2C GPIO expander,
  LCSC C465732), sharing the I2C_PWR bus (not I2C_ID -- keeps the digipot/
  ID-EEPROM bus-isolation principle intact). Address 0x20 (A0-A2 grounded),
  no conflict with the MCP4661 digipot's 0x28-range address.
- HOSTED_SPI_CLK/MOSI/MISO/CS, HOSTED_HANDSHAKE, HOSTED_DATA_READY: ESP-Hosted
  full-duplex SPI co-processor link to the interposer SoC, crossing J1 pins 43-48.
  HOSTED_RESET (pin 3, ESP32's own EN) crosses via J1 pin 86 -- host side must
  drive it open-drain so it doesn't fight R8's pull-up.
- EXP_INT: U15's interrupt output, carrier-local, not crossing J1.
